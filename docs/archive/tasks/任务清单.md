# 投标文件自动化生成任务清单（目录驱动版）

## 1. 任务总览

| ID | 任务 | 优先级 | 预计工期 | 依赖 | 验收标准 |
|---|---|---|---|---|---|
| T01 | 冻结项目目录协议 | P0 | 0.5天 | 无 | 明确 `project_dir` 输入结构 |
| T02 | 定义字段与报告Schema | P0 | 0.5天 | T01 | 完成 `normalized_payload`/`qa_report` |
| T03 | 开发招标文件解析器 | P0 | 1.5天 | T01 | 可提取章节树、条款、表格要求 |
| T04 | 开发章节规划器 | P0 | 1天 | T03 | 输出 `section_plan.json` |
| T05 | 开发资料抽取模块 | P0 | 1.5天 | T01,T02 | 输出事实字段及 `confidence/source` |
| T06 | 开发规则校验模块 | P0 | 1天 | T02,T05 | 缺失/冲突/低置信度可识别 |
| T07 | 开发可变文案生成模块 | P1 | 1天 | T03,T05 | 生成技术/商务/实施方案初稿 |
| T08 | 开发投标文档组装模块 | P0 | 2天 | T04,T06,T07 | 生成动态章节投标文件 |
| T09 | 开发QA报告导出 | P0 | 0.5天 | T06,T08 | 输出完整 `qa_report.json` |
| T10 | 打通一键CLI入口 | P0 | 0.5天 | T03~T09 | `--project-dir` 一键运行成功 |
| T11 | 样例项目测试（3-5例） | P0 | 1天 | T10 | 不同招标文件结构均可产出 |
| T12 | 文档与使用手册 | P1 | 0.5天 | T11 | 提供目录规范与运行说明 |

## 2. 分阶段计划

### Phase 1（MVP）

- T01, T02, T03, T04, T05, T06, T08, T09, T10, T11

交付物：
1. 目录驱动一键生成能力
2. 动态章节投标文件
3. 校验报告输出

### Phase 2（增强）

- T07, T12

交付物：
1. 可变文案质量提升
2. 可交接运行文档

## 3. 完成定义（DoD）

1. 仅依赖本地目录输入，不依赖对话渠道。
2. 支持招标文件结构变化下的动态生成。
3. 事实字段可追溯，固定合规模块不漂移。
4. 生成结果可复核，报告可定位问题。

## 4. 风险与关注点

1. 招标文件格式差异大，解析鲁棒性需迭代。
2. OCR质量影响事实字段准确率。
3. 动态章节映射策略初期可能出现误分类。
4. 复杂表格在docx组装时可能存在兼容问题。
