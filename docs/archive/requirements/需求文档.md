# 投标文件自动化生成需求文档

## 1. 背景与目标

基于用户提供的招标模板文档 `模板.docx`，自动生成一份投标文件（`AI生成的文件.docx` 或指定名称）。

- `模板.docx` 是唯一格式标准（章节结构、样式、表格版式）。
- `AI生成的文件.docx` 仅作为内容参考样例，不作为格式来源。

## 2. 输入与输出

### 2.1 输入

1. 模板文件：`模板.docx`
2. 用户提供材料（事实类）：
   - 供应商名称（人工输入）
   - 营业执照（文件）
   - 法人身份证明（正反面文件）
   - 其他资质/附件（财务、社保、纳税、信用、业绩等，按招标要求）
3. 用户输入的招标项目信息（可为文本、图片、PDF、聊天记录等非结构化内容）

### 2.2 输出

1. 最终投标文件：与 `模板.docx` 格式一致的 `.docx` 文件
2. 校验报告（结构化）：
   - 缺失字段
   - 低置信度字段
   - 冲突字段
   - 占位符残留字段

## 3. 业务边界

1. 事实类信息只允许“用户提供 + AI抽取”，禁止模型虚构。
2. 声明/承诺类章节使用固定模板，仅允许变量替换。
3. 方案/应答类章节允许 AI 生成初稿，但需人工复核。
4. 所有抽取字段必须可追溯：`value + confidence + source`。

## 4. 最小章节清单（v1）

1. 封面（项目名称、项目编号、供应商、日期）
2. 目录
3. 营业执照等证明文件（附件）
4. 法定代表人/单位负责人身份证明与授权书
5. 财务状况证明（附件）
6. 社保与纳税证明（附件）
7. 承诺函
8. 无行贿犯罪记录承诺函
9. 无重大违法记录承诺函
10. 信用信息查询记录和证据（附件）
11. 技术/服务应答（按项目要求）
12. 商务应答（按项目要求）
13. 实施方案（按项目要求）
14. 最终报价表（按项目要求）

## 5. 字段来源矩阵（v1）

### 5.1 项目字段

- `project_name`：AI从用户输入抽取，必填，必审
- `project_no`：AI抽取，必填，必审
- `package_no`：AI抽取，选填
- `procurement_org`：AI抽取，选填
- `agency_org`：AI抽取，选填
- `doc_date`：系统生成/用户指定，必填

### 5.2 供应商与法人字段

- `supplier_name`：人工输入 + 与营业执照交叉校验，必填，必审
- `unified_social_credit_code`：营业执照OCR抽取，必填，必审
- `company_type`：营业执照OCR抽取，选填
- `registered_address`：营业执照OCR抽取，必填，必审
- `established_date`：营业执照OCR抽取，选填
- `business_term`：营业执照OCR抽取，选填
- `legal_name`：身份证/执照抽取，必填，必审
- `legal_gender`：身份证抽取，选填
- `legal_id_no`：身份证抽取，必填，必审
- `legal_position`：AI抽取或人工补充，选填
- `contact_name`：AI抽取或人工补充，选填
- `contact_phone`：AI抽取或人工补充，选填
- `contact_email`：AI抽取或人工补充，选填
- `signatory_name`：规则推断（法人/授权人），必填，必审
- `sign_date`：系统生成/用户指定，必填

### 5.3 附件字段（用户提供）

- `license_file`：必填，必审
- `legal_id_front`：必填，必审
- `legal_id_back`：必填，必审
- `financial_proof_files`：按招标要求，必审
- `tax_proof_files`：按招标要求，必审
- `social_security_files`：按招标要求，必审
- `credit_report_files`：按招标要求，必审
- `qualification_files`：按招标要求，必审
- `performance_files`：按招标要求，必审

### 5.4 AI可生成字段（初稿）

- `technical_response_text`：按项目要求，必审
- `business_response_text`：按项目要求，必审
- `implementation_plan_text`：按项目要求，必审

### 5.5 固定模板字段（禁止自由改写）

- `承诺函`
- `无行贿犯罪记录承诺函`
- `无重大违法记录承诺函`

## 6. 自动化流程

1. 材料接入：收集模板、证照附件、用户项目输入
2. AI抽取：从非结构化输入抽取结构化字段
3. 规则校验：格式校验、字段一致性校验、必填检查
4. 文案生成：仅对允许章节生成初稿
5. 模板渲染：将字段写入 `模板.docx` 占位符，不改变样式
6. 人工复核：低置信度字段、合规章节、关键事实字段
7. 结果导出：最终 `.docx` + 校验报告

## 7. 校验与验收标准

1. 模板一致性：章节顺序、样式、表格格式与 `模板.docx` 一致
2. 字段完整性：必填字段完整，缺失项在报告中可追踪
3. 合规性：固定承诺文本未被自由改写
4. 真实性：事实字段无模型臆造
5. 可追溯性：抽取字段均有 `confidence` 与 `source`
6. 占位符清理：残留占位符数量为 0

## 8. 非功能要求

1. 敏感信息保护：身份证号等字段脱敏展示、最小化日志留存
2. 可审计：保留抽取来源、版本号、时间戳
3. 可扩展：支持新增模板与字段映射配置
4. 稳定性：同样输入应得到一致结构输出

## 9. 待确认项

1. 低置信度复核阈值（建议 `confidence < 0.90`）
2. 各项目是否都要求技术/商务应答与实施方案章节
3. 最终文件命名规范（固定命名或按项目编号命名）
