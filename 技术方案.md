# 投标文件自动化生成技术方案（目录驱动版）

## 1. 文档信息

- 文档名称：投标文件自动化生成技术方案
- 版本：v2.0
- 依据文档：`需求文档.md`
- 本期范围：不接入框架、不接入对话渠道
- 运行方式：指定资料文件夹路径，批处理生成投标文件

## 2. 目标

1. 输入一个项目资料目录，自动生成一份投标文件。
2. 投标文件结构随招标文件动态变化，不依赖固定占位模板。
3. 事实类内容来自用户资料（抽取+校验），可变文案由AI生成初稿。
4. 输出校验报告，标记缺失项、未应答条款、低置信度字段。

## 3. 输入输出规范

## 3.1 输入目录结构（约定）

`<project_dir>/`

1. `招标文件/`（必需）
- `招标文件.docx` 或 `招标文件.pdf`

2. `人工资料/`（必需）
- `供应商信息.json`（供应商名称、授权信息、报价等）
- `营业执照.*`
- `法人身份证_正.*`
- `法人身份证_反.*`

3. `附件资料/`（按招标要求）
- 财务、纳税、社保、信用、业绩、资质等附件

4. `参考资料/`（可选）
- `AI生成的文件.docx`（仅内容参考）

## 3.2 输出目录结构

`<project_dir>/output/<task_id>/`

1. `投标文件.docx`
2. `qa_report.json`
3. `normalized_payload.json`
4. `section_plan.json`（解析出的章节和生成策略）

## 4. 核心流程

1. 读取项目目录与文件索引。
2. 解析招标文件结构（目录/章节/表格/附件要求）。
3. 生成本次项目的“章节计划”：
- `FACT`：事实填充
- `LOCKED`：固定文本变量替换
- `AI_DRAFT`：AI生成初稿
- `TABLE_MAP`：表格映射填充
4. 抽取人工资料字段（OCR/规则/结构化读取）。
5. 校验字段完整性、格式和一致性。
6. 对应章节生成内容并组装为投标文件。
7. 输出质检报告与复核清单。

## 5. 人工输入 vs AI生成边界

1. 必须人工提供：
- 主体身份与证照
- 资质证明附件
- 报价与关键商务承诺数值

2. AI可自动抽取：
- 证照信息
- 招标文件项目信息与条款

3. AI可生成初稿：
- 技术/商务应答说明
- 实施方案与服务方案

4. 固定模板锁定：
- 承诺函、无行贿、无重大违法记录（仅变量替换）

## 6. 动态适配策略（招标文件变化）

1. 新增章节：自动分类并加入章节计划。
2. 新增条款：自动进入应答清单。
3. 新增附件要求：自动加入缺失清单。
4. 删除章节：自动从投标输出中移除。

## 7. 数据契约

## 7.1 `normalized_payload.json`

- 项目字段：`project_name`、`project_no`、`package_no`、`procurement_org`、`agency_org`、`doc_date`
- 主体字段：`supplier_name`、`unified_social_credit_code`、`registered_address`、`legal_name`、`legal_id_no`
- 文案字段：`technical_response_text`、`business_response_text`、`implementation_plan_text`
- 审计字段：`value`、`confidence`、`source_file`、`source_locator`

## 7.2 `qa_report.json`

- `missing_required_fields`
- `missing_attachments`
- `unanswered_clauses`
- `low_confidence_fields`
- `conflict_fields`
- `manual_review_required_sections`

## 9. 验收标准（目录驱动MVP）

1. 给定 `project_dir` 可一键产出投标文件与报告。
2. 投标文件章节结构可随招标文件变化自动调整。
3. 事实字段无臆造，缺失项在报告中可定位。
4. 固定承诺章节未被自由改写。
5. 报告含未应答条款与必审章节列表。
